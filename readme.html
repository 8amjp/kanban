Kanban board

Kanban boardによるタスク管理

使い方

準備

  1. 「99_個人フォルダ」にある自分のフォルダ内に、 「#ToDo」「#Doing」「#Done」フォルダを作成します。
  2. 「#ToDo」フォルダ内に「#Someday」フォルダを作成します。
  3. 「#Doing」フォルダ内に「#Waiting」フォルダを作成します。

表示

  1. 「準備」で作成した各フォルダ内に、さらにフォルダを作成すると、Kanban board上に「タスク」として表示されます。 (以降、これを「カンバン」と標記します)
  2. カンバンをクリックすると、詳細を表示します。
  3. タスクフォルダ内に「memo」という名前(拡張子なし)のファイルを作成すると、記載内容が「メモ」欄に表示されます。
  4. タスクフォルダ内に「style」という名前(拡張子なし)のファイルを作成し、 スタイルをJSON形式で記述すると、カンバンのスタイルを変更できます。

    例:

    {"background-color":"pink"}
    • タスクフォルダ内に「style」ファイルがない場合、上位のフォルダをさかのぼって検索します。 どの階層にもない場合はデフォルトのスタイルになります。

実践(GTDによるタスク管理)

収集

  1. 指示された仕事、思いついた課題などなど、とるべき行動をすべてリストに書き出します。
  2. 「タスク名(とるべき行動)」をフォルダ名としたフォルダを必要個数分作成します。
  3. 下記のルールで仕分けを行います。

処理

  1. 2分以内に完了するタスクはその場で完了させ、「#Done」に移動します。
  2. すぐに対応すべきタスクは「#ToDo」に移動します。
  3. すぐに対応する必要のないタスクは「#Someday」に移動します。
  4. 他の人に対応を任せたもの、他の要因により進展しないタスクは「#Waiting」に移動します。
  5. 特定の日時まで行動不要なタスクは、スケジューラに登録の上、「#Waiting」に移動します。

実行

仕分けが終わったら、下記のルールでタスクを片づけていきます。

  1. 「#ToDo」内のタスクから1つ「#Doing」に移動し、取り掛かります。
  2. 完了したら「#Done」フォルダに移動し、次に取り掛かるタスクを「#ToDo」から「#Doing」に移動します。

利点

  • 「あれもこれもやらなければ...」という漠然とした不安を軽減します。
  • 参考資料を全部デスクトップに置いている人！デスクトップがスッキリします。
  • 「#Done」フォルダの中身ををさらに完了日(半期)毎に分ければ、成果を振り返るときに楽です。
